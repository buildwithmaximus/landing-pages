// what is claude code

Not a chatbot. An agent that does things.

Claude Code runs in your terminal and has direct access to your files, your git history, and your shell. It understands context. It takes action.

Reads your entire codebase

Claude Code understands your project structure, existing patterns, dependencies, and conventions before it writes a single line.

Edits files directly

No copy-pasting. Claude Code makes targeted edits to your files — one change at a time, or across multiple files in a single step.

Runs shell commands

It can install packages, run tests, start servers, and execute scripts. It sees the output and adapts if something goes wrong.

Persistent across sessions

With a CLAUDE.md file, Claude Code carries your project context, preferences, and conventions into every session automatically.

// setup guide

From zero to running in 5 minutes

You need Node.js and a Claude subscription. That's it.

01

Install Claude Code

Install the CLI globally via npm. Requires Node.js 18+.

npm install -g @anthropic-ai/claude-code
# Verify installation
claude --version
02

Authenticate

Sign in with your Anthropic account. Claude Pro or Max subscription required.

claude auth login
# Opens browser → sign in with claude.ai account
✓ Authenticated as you@example.com
03

Open your project

Navigate to any project folder and launch Claude Code. It will read the codebase automatically.

cd ~/your-project
claude
✻ Initialized. Reading codebase...
04

Create your CLAUDE.md

This file gives Claude persistent context about your project. It loads automatically every session.

# CLAUDE.md — project context for Claude Code

## Project
Next.js 14 app using TypeScript + Tailwind.
API routes in /app/api/, components in /components/

## Rules
- Always use TypeScript, never plain JS
- Run `npm test` after every change
- Never modify .env files directly
05

Give your first instruction

Talk to it like a senior developer. Be specific about what you want. It will ask if it needs clarification.

# Examples of good first instructions

"Add input validation to the signup form with error messages"
"Refactor the UserCard component to use our design system"
"Find all console.log statements and remove them before production"
"Write unit tests for the auth module"
// key files

The files that make Claude Code yours

Three files. Master these and you have a fully personalized AI agent that works exactly the way you want.

CLAUDE.md Required

Project-level context. Describes your tech stack, folder structure, conventions, and rules. Lives in the project root. Loaded every session automatically.

.claude/CLAUDE.md Global

Your personal global context. Preferences, style rules, and defaults that apply across every project. Think of it as your personal developer profile.

.claude/skills/ Skills

Reusable slash-command workflows. Each folder is a skill — type /skill-name and Claude runs it. Perfect for repeated tasks.

.mcp.json Integrations

Model Context Protocol config. Connects Claude Code to external tools — databases, APIs, Notion, GitHub, Slack — expanding what it can access and control.

// pro tips

Get more out of every session

Small habits that dramatically improve the quality of what Claude Code produces.

Be specific, not vague

"Add dark mode" is vague. "Add a dark mode toggle in the top-right of the navbar that saves preference to localStorage and respects prefers-color-scheme" is actionable. The more context you give, the better the output.

Keep CLAUDE.md updated

Every time you establish a new convention or catch a mistake, add it to CLAUDE.md. It compounds — the more you teach it, the less you need to explain. Treat it like a living document.

Review before accepting

Claude Code shows you exactly what it plans to change before applying. Always read the diff. It's usually right — but you're the decision-maker, not the agent.

Build skills for repetitive tasks

If you explain a workflow more than twice, turn it into a skill file. Store it in .claude/skills/. From that point on, one slash command runs the whole thing reliably.

Keep secrets out of the workspace

Never hardcode API keys or tokens in files Claude Code can read. Use .env files and reference them by variable name. What Claude reads, Claude can output.
