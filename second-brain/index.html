How it works

The inbox workflow

Everything flows through the inbox. You send, Maximus processes, the vault grows. You never edit files directly.

1

Send raw material via Telegram

Drop a URL, paste text, send an image, share an idea. Any format works. The message is your trigger.

"Add this to my vault: https://example.com/article"
2

Maximus files it to Inbox/raw/

Raw content is captured as-is into Inbox/raw/. Nothing is lost. The original source is preserved as the audit trail.

3

Content is compiled into a vault note

Maximus reads the raw file, identifies the right folder, writes a structured note with frontmatter, tags, and wikilinks. No invented entities — only real ones from the vault.

4

Raw file moves to Inbox/processed/

The source file is archived — never deleted. It becomes the audit trail showing where vault content came from.

5

Natan pulls in Obsidian

Cmd+P → Obsidian Git: Pull. The new note appears in the vault, connected to the graph.

Cmd+P → "Obsidian Git: Pull"

For AI agents

Agent quickstart

If you are an AI agent loading this vault, follow these steps. Do not skip step 1.

Step 1 — Read AGENT_CONTEXT.md (mandatory)

# GitHub API — read the master briefing GET https://api.github.com/repos/Natan-Yago/obsidian-second-brain/contents/AGENT_CONTEXT.md Authorization: Bearer [your-token] # Decode base64 content field → read top to bottom

Step 2 — Check today's daily note

GET https://api.github.com/repos/Natan-Yago/obsidian-second-brain/contents/Daily/YYYY-MM-DD.md # Shows what happened today, what's in progress, what Natan is focused on

Step 3 — Write to the vault (follow the rules)

PUT https://api.github.com/repos/Natan-Yago/obsidian-second-brain/contents/PATH Body: { "message": "commit message", "content": "[base64 encoded markdown]", "sha": "[sha of existing file if updating]" } # Always include frontmatter. Always use [[wikilinks]]. Never invent entities.

Conventions

Rules every agent must follow

These rules ensure the vault stays consistent and trustworthy over time.

Always do

  • Read AGENT_CONTEXT.md before any action
  • Include full frontmatter on every note you create
  • Use [[wikilinks]] on first mention of any entity
  • File content to the right folder (see vault map)
  • Preserve Inbox/processed/ as audit trail
  • Use only the 22 approved tags from §4 taxonomy
  • Ask Natan via Telegram if anything is ambiguous

Never do

  • Edit Maximus/Identity/ without telling Natan
  • Publish Personal/, Daily/, or AGENT_CONTEXT.md
  • Invent people, projects, or resources that don't exist
  • Add tags outside the approved taxonomy
  • Delete files from Inbox/processed/
  • Use filler, sycophancy, or placeholder content
  • Assume — ask if unclear
